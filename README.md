# IntensiveTasksProject
## Практическая часть интенсива по Java в Aston.  

#### module 2
`Добавить в user-service поддержку Spring и разработать API, которое позволит управлять данными.`
- [ ] Использовать необходимые модули spring(boot, web, data etc).
- [ ] Реализовать api для получения, создания, обновления и удаления юзера. Важно, entity не должен возвращаться из контроллера, необходимо использовать dto.
- [ ] Заменить Hibernate на Spring data JPA.
- [ ] Написать тесты для API(можно делать это при помощи mockMvc или других средств)  

#### module 1
`Написать юнит-тесты и интеграционные тесты для user-service.`
- [x] Использовать JUnit 5, Mockito и Testcontainers.
- [x] Для тестирования DAO-слоя написать интеграционные тесты с использованием Testcontainers.

#### module 0  
`Разработать консольное приложение(user-service) на Java, использующее Hibernate для взаимодействия с PostgreSQL, без использования Spring.
Приложение должно поддерживать базовые операции CRUD (Create, Read, Update, Delete) над сущностью User.`
- [x] Использовать Hibernate в качестве ORM.  
- [x] База данных — PostgreSQL.
- [x] Настроить Hibernate без Spring, используя hibernate.cfg.xml или properties-файл.
- [x] Реализовать CRUD-операции для сущности User (создание, чтение, обновление, удаление), которая состоит из полей: id, name, email, age, created_at.
- [x] Использовать консольный интерфейс для взаимодействия с пользователем.
- [x] Использовать Maven для управления зависимостями.
- [x] Настроить логирование.
- [x] Настроить транзакционность для операций с базой данных.
- [x] Использовать DAO-паттерн для отделения логики работы с БД.
- [x] Обработать возможные исключения, связанные с Hibernate и PostgreSQL
- [x] Использовать паттерн Proxy
